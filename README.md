# Clock On/Off Web App

A simple, mobile-friendly web app for restaurant staff to record their start and finish work times. Built with vanilla JavaScript, Tailwind CSS, and Supabase.

## 🚀 Quick Start

### 1. Create Supabase Project

1. Go to [Supabase](https://supabase.com) and create a free account
2. Create a new project (free tier is sufficient)
3. Wait for the project to initialize (2-3 minutes)

### 2. Set Up Database

1. Open your Supabase project dashboard
2. Go to **SQL Editor** (left sidebar)
3. Run the migration files in order:

```sql
-- First: Create tables
-- Copy contents of supabase/001-initial-schema.sql and run

-- Second: Enable Row-Level Security
-- Copy contents of supabase/002-rls-policies.sql and run

-- Third: Create functions
-- Copy contents of supabase/003-functions.sql and run
```

### 3. Configure Credentials

1. In Supabase dashboard, go to **Settings** > **API**
2. Copy the **Project URL** and **anon public** key
3. Open `db.js` in this project
4. Replace the placeholder values at the top:

```javascript
const SUPABASE_URL = 'https://your-project-id.supabase.co';
const SUPABASE_ANON_KEY = 'your-anon-key-here';
```

### 4. Deploy to GitHub Pages

1. Create a new GitHub repository
2. Push all files to the repository
3. Go to **Settings** > **Pages**
4. Select **Deploy from a branch**
5. Choose **main** branch and **/ (root)**
6. Click **Save**
7. Your site will be live at `https://your-username.github.io/repo-name`

### 5. Create Admin User

1. Open your deployed site
2. Go to Supabase **SQL Editor**
3. Run this query to generate a hashed PIN:

```sql
SELECT crypt('1234', gen_salt('bf', 8));
```

4. Copy the result (it will look like `$2a$08$...`)
5. Go to Supabase **Table Editor**
6. Open the **staff** table
7. Click **Insert** and add an admin user:

| Field | Value |
|-------|-------|
| name | Your Name |
| pin_hash | Paste the hash from step 4 |
| role | `admin` |
| active | `true` |

**Important:** PINs are always hashed server-side using bcrypt. The `pin_hash` field must contain a bcrypt hash generated by the database.

## 📱 Features

- **PIN Authentication**: Simple 4-6 digit PIN for staff login
- **Clock On/Off**: Record start and finish times with ±10 minute adjustment window
- **Shift History**: Staff can view their recent shifts
- **Admin Panel**: Manage staff, view all timesheets, export CSV
- **Audit Trail**: All admin adjustments are logged
- **Mobile-First**: Works on iOS Safari and Android Chrome
- **No Build Step**: Pure static files, deploy by git push

## 🔒 Security

### PIN Hashing
PINs are hashed server-side using bcrypt (pgcrypto). The client sends the plain PIN over HTTPS to the RPC; the database hashes and compares it. No client-side hashing exists.

### Row-Level Security (RLS)
Supabase RLS policies ensure:
- Staff can only view/edit their own shifts
- Admins can access all records
- Audit log is immutable

### Time Window Enforcement
The ±10 minute adjustment window is enforced:
- **Client-side**: JavaScript validates before submission
- **Server-side**: PostgreSQL functions reject invalid times

This prevents API bypass even if someone modifies the JavaScript.

## 📊 Database Schema

### Tables

**staff**
- `id` (UUID, primary key)
- `name` (text)
- `pin` (text, unique, hashed)
- `role` (text: 'staff' or 'admin')
- `active` (boolean)
- `created_at` (timestamp)

**shifts**
- `id` (UUID, primary key)
- `staff_id` (UUID, foreign key)
- `clock_in` (timestamp)
- `clock_out` (timestamp, nullable)
- `clock_in_adjusted` (boolean)
- `clock_out_adjusted` (boolean)
- `notes` (text)
- `created_at` (timestamp)

**audit_log**
- `id` (UUID, primary key)
- `admin_id` (UUID, foreign key)
- `target_staff_id` (UUID, foreign key)
- `shift_id` (UUID, foreign key)
- `action` (text)
- `old_value` (jsonb)
- `new_value` (jsonb)
- `created_at` (timestamp)

## ⚠️ Known Limitations

1. **No Password Recovery**: If a staff member forgets their PIN, an admin must reset it
2. **Single Location**: Designed for single restaurant use (no multi-location support)
3. **No Payroll**: Does not calculate wages or export to payroll systems
4. **Client-Side Session**: Sessions stored in localStorage (cleared if cache is cleared)

## 📝 Out of Scope

The following are intentionally NOT included:

- Payroll calculations
- Schedule/roster management
- Photo/geo verification
- Multi-location support
- Email notifications
- Push notifications

## 🆘 Troubleshooting

### "Failed to get server time"
- Check Supabase credentials in `db.js`
- Ensure Supabase project is active
- Check browser console for CORS errors

### "Invalid PIN"
- Verify the PIN hash was generated with `SELECT crypt('1234', gen_salt('bf', 8));` in Supabase SQL Editor
- Check that staff record has `active = true`
- Clear browser cache and try again

### "Time must be within ±10 minutes"
- This is expected behavior
- Server time is used, not device time
- Staff must clock on/off within the grace window

### CSV export not working
- Ensure popup blocker is disabled
- Check browser console for errors
- Try a different browser
